CSS especificity

Se existem duas ou mais regras css que apontam para o mesmo elemento, o seletor com a maior especificidade vai "ganhar", e sua declaração de estilo será aplicada para o elemento html

Pense a especificidade como um score/rank que determina qual declaração de estilo é uktimamente aplicada para um elemento.

Specificity hierarchy

Todo seletor CSS tem seu lugat na hierarquia de specificidade

existem 4 categorias que definem o level de especificidade para um seletor

  1. Inline styles - Exemplo: h1 style="color:pink;"
  2. IDs - Exemplo: #navbar
  3. Classes, pseudo-classes, seletores de atributos - Exemplo: .test, :hover, [href]
  4. Elementos e pseudo-elementos - Exemplo: h1, ::before

Como calcular a especificidade

começe com 0, adicione 100 para cada valor de ID, adicione 10 para valores de classe(ou pseudo-classe ou seletores de atributos), adicione 1 para cada seletor de elemento ou pseudo-elemento.

Seletor Valor de especificidade calculo
* 0 0(seletor universal é ignorado)
p 1 1
.test 10 10
#demo 100 100
p style="color:pink;" 1000 1000
p.test 11 1 + 10
p.test1.test2 21 1 + 10 + 10
p#demo 101 1 + 100
#navbar p#demo 201 100 + 1 + 100

Outras regras de especificidade

IDS tem mais especificidade que seletores de atributos

div#a
div[id=a]

Seletores contextuais são mais especificos que seletores de elemento único

Um seletor de classe vence qualquer número de seletores de elemento
